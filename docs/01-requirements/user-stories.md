# User Stories — Daily Commit

This document lists initial MVP user stories.

The stories are intentionally written for a single-user product, but should remain clear enough for future generalization.

---

## Objective management

### US001 — Create objective

As the founder, I want to create reusable objectives so that I can select them when configuring a day.

### US002 — Edit objective

As the founder, I want to edit reusable objectives so that I can refine names, targets, units, and defaults.

### US003 — Deactivate objective

As the founder, I want to deactivate objectives so that outdated or temporarily irrelevant objectives do not appear during day configuration.

---

## Day configuration

### US004 — Configure day manually

As the founder, I want to manually configure a day so that I can decide what I am committing to today.

### US005 — Assign base objective weights

As the founder, I want to assign weights to base objectives so that more important objectives have more score impact.

### US006 — Validate base total

As the founder, I want the app to validate that base objectives sum to 100% so that the daily score remains clear and honest.

### US007 — Require minimum base objectives

As the founder, I want the app to require at least 3 base objectives so that a weak configuration cannot represent a full day.

### US008 — Add bonus objectives

As the founder, I want to add optional bonus objectives so that extra effort can compensate for missed base points without changing the original commitment.

### US009 — Mark day as excluded

As the founder, I want to mark a day as excluded so that intentional rest days do not count toward weekly performance.

### US010 — Edit active day

As the founder, I want to edit an active day when needed so that I can correct mistakes or adjust reality, while being reminded that I am changing the original commitment.

---

## Daily execution

### US011 — See today's commitment

As the founder, I want to see today's objectives immediately so that I do not have to remember the commitment.

### US012 — Mark binary objective completed

As the founder, I want to mark binary objectives as completed from Today so that updating progress is fast.

### US013 — Update numeric progress

As the founder, I want to update numeric objectives from Today so that partial progress is reflected in the score.

### US014 — See live score

As the founder, I want the score to update immediately after progress changes so that I get fast feedback.

### US015 — See incomplete base objectives

As the founder, I want incomplete base objectives to remain visible even if bonus raises the score so that I do not lose sight of the original commitment.

---

## Day closure

### US016 — Auto-close on base completion

As the founder, I want the day to close automatically when all base objectives are complete so that the system recognizes a fulfilled commitment.

### US017 — Auto-close at midnight

As the founder, I want the day to close automatically at midnight so that every active day ends with a final score.

---

## Weekly review

### US018 — View weekly performance

As the founder, I want to see weekly performance so that I understand how scored days went.

### US019 — View weekly consistency

As the founder, I want to see scored, excluded, and unconfigured days separately so that the weekly score is not misleading.

### US020 — Week starts on Monday

As the founder, I want weeks to start on Monday so that the product matches my local weekly rhythm.
