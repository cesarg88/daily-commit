# Development Horizons

PEOS uses five horizons to keep product intent, technical direction, delivery, and approval separate.

## 1. Product Vision

The CEO defines the product problem, audience, value proposition, principles, and non-goals.

## 2. Technical Strategy

The CTO defines technical direction, architectural boundaries, quality expectations, and major tradeoffs.

The Lead Orchestrator and specialized agents may produce proposals during this horizon, but CEO/CTO approval is required before implementation starts.

No code should be written before the implementation proposal is approved.

## 3. Delivery

Specialized agents implement the approved proposal in small, focused changes. The Lead Orchestrator coordinates scope, sequencing, and handoffs.

## 4. Technical Review

The CTO reviews architecture, maintainability, tests, risk, and technical fit against the approved strategy.

## 5. Merge

The CEO and CTO approve merge when product behavior and technical quality meet the agreed bar.
